# Нормализация цветов ручек в модалке «Выбор ручки»

## Получившиеся нормализованные цвета

После нормализации в фильтре «Цвет» отображаются следующие варианты (одно название на группу):

| Нормализованное название | Варианты из названий ручек, которые к нему приводятся |
|-------------------------|-------------------------------------------------------|
| **Хром** | ХРОМ-МАТ, ХРОМ+МАТ, хром мат, хроммат, хром/мат, хром матовый, xpom-mat, XPOM-MAT и т.п. |
| **Белый** | белый, белая |
| **Черный** | черный, черная |
| **Графит** | графит, разное графит |
| **Графит браш** | графит браш |
| **Никель** | никель |
| **Сатиновый никель** | мат. сатиновый никель, мат сатиновый никель, сатиновый никель |
| **Золото** | золото, золотой |
| **Латунь** | латунь |

Правила заданы в `lib/handle-color-normalize.ts` (массив `ALIASES`).

## Не удалось нормализовать

Варианты, для которых **нет** правила в `ALIASES`, показываются в фильтре как есть (с капитализацией первой буквы). Они не объединяются с другими цветами.

При открытии модалки выбора ручки в консоли браузера выводится сообщение:
`Ручки: цвета, которые не удалось нормализовать (остаются как есть): [...]`

Если в списке появляются новые сырые значения — их можно добавить в `lib/handle-color-normalize.ts` (новые правила в `ALIASES`).

## Фильтры в модалке

- **Цвет** и **Цена** — два отдельных фильтра.
- Оба работают **одновременно**: отображаются ручки, подходящие и по цвету, и по выбранным диапазонам цены.
- По каждому фильтру можно выбрать несколько значений (несколько цветов и/или несколько ценовых диапазонов).
