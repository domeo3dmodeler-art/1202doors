# Хосты ВМ и пайплайн деплоя

## Канонический пайплайн

```
Локальная разработка  →  Тестовая ВМ (178.154.244.83)  →  Рабочая ВМ (158.160.69.237)
```

1. **Разрабатываем локально** (`npm run dev`).
2. **Деплоим на тестовую** — проверяем, что всё работает.
3. **Переносим с тестовой на рабочую** — при деплое на рабочую **не трогаем пользователей, документы и заказы**.

## Быстрые команды

### Деплой на тестовую ВМ (по умолчанию)

```powershell
.\scripts\deploy-to-test-vm.ps1              # сборка + rsync (дельта)
.\scripts\deploy-to-test-vm.ps1 -SkipBuild   # rsync без пересборки
.\scripts\deploy-to-test-vm.ps1 -NoRsync     # полный архив (fallback)
```

### Деплой на рабочую ВМ

```powershell
.\scripts\deploy-to-prod-vm.ps1              # сборка + rsync (дельта)
.\scripts\deploy-to-prod-vm.ps1 -SkipBuild   # rsync без пересборки
```

Скрипт `deploy-to-prod-vm.ps1` требует подтверждения перед деплоем. `.env` и `public/uploads` на ВМ не затрагиваются.

### Перенос БД и фото с рабочей на тестовую

```powershell
.\scripts\sync-prod-to-test-vm.ps1             # БД + фото
.\scripts\sync-prod-to-test-vm.ps1 -SkipPhotos # только БД
```

### Переключение окружения вручную

```powershell
# Тестовая ВМ (по умолчанию для всех скриптов):
.\scripts\set-test-vm-env.ps1

# Рабочая ВМ:
.\scripts\set-prod-vm-env.ps1
```
