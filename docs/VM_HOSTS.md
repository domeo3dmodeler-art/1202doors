## Переключение на тестовую ВМ

В PowerShell перед запуском скриптов деплоя/синхронизации:

```powershell
.\scripts\set-test-vm-env.ps1
# либо вручную:
$env:1002DOORS_STAGING_HOST = "ubuntu@130.193.62.116"
$env:1002DOORS_SSH_KEY = "C:\Users\petr2\testdoors\ssh-key-1773299302859\id_ed25519"   # или путь к ключу в этой папке
```

Чтобы снова использовать рабочую машину:

```powershell
$env:1002DOORS_STAGING_HOST = "ubuntu@89.169.181.191"
$env:1002DOORS_SSH_KEY = "C:\Users\petr2\.ssh\ssh-key-1771526730154\ssh-key-1771526730154"
# или просто удалить переменные — скрипты подхватят рабочую по умолчанию
Remove-Item Env:1002DOORS_STAGING_HOST -ErrorAction SilentlyContinue
Remove-Item Env:1002DOORS_SSH_KEY -ErrorAction SilentlyContinue
```
